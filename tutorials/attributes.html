Phase 02: Attributes & Self

The State of an Object

In Python, Attributes are the variables that belong to an object. They represent the "State"—the data that differentiates one instance from another. While the class defines what kind of data can be stored, the attributes hold the specific values. Mastering attributes is the difference between writing a generic script and building a powerful, data-driven system where objects "remember" who they are and what they are currently doing.

Demystifying 'Self'

The most common point of confusion for Python beginners is the self parameter. In a practical sense, self is not a keyword, but a conventional name for the first argument of any method. It serves as a pointer to the specific memory address of the object being manipulated. Without self, Python would have no way to distinguish which object's data to retrieve or modify when a method is called by multiple instances.

Instance Attributes

Defined inside __init__ with the self prefix. These are unique to every object you create, allowing "Object A" to have a different name or value than "Object B".

Class Attributes

Defined directly under the class header. These are shared across all instances of that class—ideal for constants or shared configurations.

class Drone:
    # Class Attribute (shared)
    category = "UAV"

    def __init__(self, model: str, battery: int):
        # Instance Attributes (unique)
        self.model = model
        self.battery = battery

    def status(self):
        # 'self' allows access to unique instance data
        return f"{self.model}: {self.battery}% power remaining."

alpha = Drone("Phantom", 85)
beta = Drone("Mavic", 42)

print(alpha.status()) # Unique to alpha
print(beta.status())  # Unique to beta

Attribute Lifecycle

Attributes in Python are highly dynamic. Unlike more rigid languages, Python allows you to add or modify attributes at runtime. However, professional architecture discourages this "on-the-fly" modification. In a production environment, all attributes should be explicitly declared within the constructor to maintain code predictability and ensure that your object's structure remains transparent to other developers.
