Phase 01: Blueprints

The Class vs. Object Distinction

In the Python ecosystem, a Class is a structural definition—a blueprint that outlines what data an entity will hold and what actions it can perform. It exists in your code as a template but occupies no specific space in memory until it is invoked. An Object, or Instance, is the living manifestation of that blueprint. When you instantiate a class, Python allocates memory to store that specific version's data, allowing multiple objects to exist simultaneously while remaining entirely independent of one another.

The __init__ Method

The constructor. This is where you define the initial state of your object. In modern Python, we prioritize setting only essential data here to keep our objects lean and focused.

The Self Keyword

Self is the explicit reference to the instance. It is the hook that allows methods to access the data stored within that specific object rather than the general class scope.

# Defining the blueprint
class Architect:
    def __init__(self, name: str, specialty: str):
        self.name = name
        self.specialty = specialty

    def design(self) -> str:
        return f"{self.name} is creating a {self.specialty} plan."

# Creating distinct objects (Instances)
lead_dev = Architect("Glenn", "System Design")
junior_dev = Architect("Alex", "Frontend")

print(lead_dev.design()) # Glenn is creating a System Design plan.

Modern Architectural Thinking

While traditional OOP focuses heavily on deep inheritance, modern Python architecture favors Composition. This means building complex objects by combining smaller, specialized classes rather than creating a massive hierarchy of parent and child classes. This approach reduces coupling and makes your system significantly more resilient to changes in requirements as your project evolves.
