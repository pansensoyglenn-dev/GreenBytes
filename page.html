The Power of Encapsulation

Architecture for a World Flooded with Data.

Course Syllabus

The Logic of Isolation

In this "Data Flood," the primary failure of software is a lack of control. Encapsulation wraps data and methods into a single unit to hide internal states.

Why Python for the Architect?

Python allows you to focus entirely on System Design. It treats encapsulation not just as a keyword, but as a conceptual boundary. By using Python, we abstract the 'How' so we can master the 'Why'.

Bank Account Management

class BankAccount:
    def __init__(self, account_holder, initial_balance=0):
        self.__account_holder = account_holder
        self.__balance = initial_balance

    def deposit(self):
        amount = float(input("Enter amount: "))
        if amount > 0: self.__balance += amount
        print(f"Balance: ${self.__balance}")

    def withdraw(self):
        amount = float(input("Enter amount: "))
        if 0 < amount <= self.__balance: self.__balance -= amount
        print(f"Balance: ${self.__balance}")
The Architect's Logic: The Why

The Thought: Money is too sensitive to leave exposed.
The Output: __balance ensures money only moves through verified logic gates.

Inventory System

class InventoryItem:
    def __init__(self, item_name, initial_stock=0, price=0.0):
        self.__item_name = item_name
        self.__stock = initial_stock
        self.__price = price

    def set_price(self, new_price):
        if new_price > 0: self.__price = new_price
        else: print("Error: Negative pricing blocked.")
The Architect's Logic: The Why

The Thought: Business integrity relies on valid pricing.
The Output: set_price acts as a shield against catastrophic financial errors.

Employee HR Management

class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.__salary = salary

    def update_salary(self, amount):
        if amount > self.__salary: self.__salary = amount
        else: print("Error: Cannot decrease base salary.")
The Architect's Logic: The Why

The Thought: Personal data requires privacy and strict update rules.
The Output: Prevents salary tampering while allowing legitimate raises.

Library Manager

class Book:
    def __init__(self, title):
        self.title = title
        self.__is_checked_out = False

    def toggle_checkout(self):
        self.__is_checked_out = not self.__is_checked_out
        status = "Out" if self.__is_checked_out else "In"
        print(f"{self.title} is now {status}")
The Architect's Logic: The Why

The Thought: Asset status should not be toggled manually by external modules.
The Output: Centralizes the "Checkout" logic to avoid inventory sync errors.

Student Grade Tracker

class Student:
    def __init__(self, name):
        self.name = name
        self.__grades = []

    def add_grade(self, grade):
        if 0 <= grade <= 100: self.__grades.append(grade)
        else: print("Invalid Grade.")

    def get_average(self):
        return sum(self.__grades) / len(self.__grades) if self.__grades else 0
The Architect's Logic: The Why

The Thought: Raw grade lists are vulnerable to illegal insertions.
The Output: Protects the GPA calculation from outlier data or 100+ point errors.

Patient Records

class Patient:
    def __init__(self, name, medical_history):
        self.name = name
        self.__history = medical_history

    def get_history(self, access_level):
        if access_level == "DOCTOR": return self.__history
        return "Access Denied: Highly Confidential."
The Architect's Logic: The Why

The Thought: Data exposure in Healthcare is a legal liability.
The Output: Creates a permission-based wall between data and users.

Smart Energy Monitor

class SmartMeter:
    def __init__(self):
        self.__usage_kw = 0.0

    def log_usage(self, kw):
        if kw > 0: self.__usage_kw += kw

    def get_bill(self):
        return f"Current Bill: ${self.__usage_kw * 0.12:.2f}"
The Architect's Logic: The Why

The Thought: Meters shouldn't be "rewound" or manually reset.
The Output: Ensures that usage data is additive only, preventing billing fraud.

E-Commerce Cart

class ShoppingCart:
    def __init__(self):
        self.__items = {}

    def add_item(self, item, price):
        self.__items[item] = price

    def total_price(self):
        return sum(self.__items.values())
The Architect's Logic: The Why

The Thought: Cart data must be isolated from the checkout total calculation.
The Output: Protects the dictionary of items from accidental deletion during runtime.

Secure Authentication

class User:
    def __init__(self, username, password):
        self.username = username
        self.__password = password

    def check_password(self, attempt):
        return attempt == self.__password
The Architect's Logic: The Why

The Thought: Passwords should never be retrievable, only verifiable.
The Output: No get_password method exists, keeping the secret hidden forever.

Autonomous Sensors

class Sensor:
    def __init__(self):
        self.__reading = 0.0

    def update_sensor(self, val):
        if -50.0 <= val <= 150.0: self.__reading = val
        else: print("Sensor Hardware Error: Out of Bounds.")
The Architect's Logic: The Why

The Thought: Hardware data can be erratic or faulty.
The Output: Provides a software "Safe Zone" to filter out physical glitches.

Data Ethics & Principles

In a world flooded with data, we prioritize Encapsulation over Exposure.

Building software is not about what you allow the user to do; it is about what you prevent the system from doing to itself. True power lies in the boundaries we draw—the lines where data ends and security begins.
